# Programming Technique I (Year 1, Semester 1)

## Overview
Welcome to the `programming_techniques_I` directory of my ePortfolio. This folder contains the coursework, assignments, and projects completed during my first semester. The module serves as a rigorous introduction to computer programming, focusing on procedural programming concepts, algorithmic logic, and problem-solving using C++. 

## Objectives
* To understand the fundamental principles of programming and software design.
* To develop logical thinking and algorithmic problem-solving skills.
* To master basic C++ syntax, including control structures, functions, arrays, and pointers.
* To practice writing, debugging, and compiling functional code to solve real-world problems.

## What I Did
Throughout this module, I transitioned from writing simple standard output statements to developing comprehensive console-based applications. I translated problem statements into pseudocode and flowcharts, which I then coded into functional C++ programs. This involved heavy debugging, managing variable scopes, manipulating array data, and structuring code cleanly using modular functions. For the final project, I collaborated with peers to design a cohesive system that required integrating various foundational concepts into a single, executable solution.

## Key Learning Outcomes
1. **Algorithmic Thinking:** Gained the ability to break down complex problems into logical, step-by-step algorithms.
2. **C++ Fundamentals:** Acquired a strong grasp of data types, operators, and control structures.
3. **Memory and Data Handling:** Learned how to manage data using arrays and gained introductory exposure to pointers and memory management.
4. **Debugging and Troubleshooting:** Developed the patience and analytical skills to trace logic errors, resolve syntax issues, and optimize code execution.

## Reflection
Looking back, the rigorous syntax and memory management concepts learned in this module were crucial building blocks. While my current focus involves building data pipelines, writing Python scripts to track market assets, and structuring SQL/NoSQL databases, the algorithmic discipline required in C++ set the foundation for it all. Understanding how arrays and functions operate at a lower level makes it much easier to conceptualize how data is processed and stored in larger cloud environments and analytics platforms today. The transition from these early procedural concepts to modern data engineering frameworks feels like a natural evolution of the problem-solving skills developed here.

## Possible Improvements
If I were to revisit these early projects, I would focus more heavily on writing cleaner, self-documenting code. In the beginning, my focus was purely on getting the code to compile and run correctly. Now, I understand the importance of comprehensive commenting, consistent naming conventions, and utilizing version control (Git) more effectively from day one. Additionally, I would try to optimize my loops and conditional logic to be more resource-efficient.
