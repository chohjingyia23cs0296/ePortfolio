# Computer Organization and Architecture (Year 1, Semester 2)

## Overview
Welcome to the `Computer Organization and Architecture` directory of my ePortfolio. This folder documents my coursework and assignments from the second semester. This module explores the fundamental hardware components of computer systems, bridging the gap between high-level software programming and low-level physical execution. 

## Objectives
* To understand the internal structure, components, and functional operation of a modern central processing unit (CPU).
* To learn how data is represented, stored, and manipulated at the machine level (binary, hexadecimal, floating-point formats).
* To explore memory hierarchies, including cache, main memory (RAM), and secondary storage, and how they impact system performance.
* To gain practical exposure to assembly language programming and instruction set architectures.

## What I Did
Throughout this module, I studied the physical and logical foundations of computing systems. I started by mastering data representation and boolean algebra before moving on to structural designs like logic gates and multiplexers. I also wrote foundational assembly language code, which required manually managing registers and memory addresses to execute basic algorithms. Additionally, I analyzed system performance metrics, calculating clock cycles, instruction execution times, and cache hit rates to understand what makes a computer run efficiently.

## Key Learning Outcomes
1. **Hardware-Software Interface:** Developed a clear understanding of how high-level code is ultimately compiled down into machine-level instructions executed by the CPU.
2. **Memory Management:** Learned how memory hierarchies operate, highlighting the critical performance differences between cache, RAM, and disk storage.
3. **Low-Level Programming:** Gained practical experience writing assembly language, fostering a deep appreciation for the abstractions provided by modern programming languages.
4. **System Optimization:** Acquired the ability to evaluate compute performance based on clock speed, datapath design, and instruction pipelining.

## Reflection
Studying computer architecture provided the essential hardware foundation that supports my ongoing progression in Data Engineering. Understanding how a CPU processes instructions and how memory hierarchies operate has been incredibly useful when configuring cloud infrastructure on platforms like AWS, Azure, and Google Cloud. It provides a much clearer perspective on the actual compute and storage resources I am allocating when building scalable data pipelines or provisioning environments for database management. Knowing the physical mechanics and limitations of hardware makes it much easier to optimize heavy SQL and NoSQL queries to run efficiently at scale.

## Possible Improvements
* **Connecting to Cloud Architecture:** If I were to revisit these assignments, I would try to draw more direct comparisons between traditional CPU/memory architectures and modern distributed cloud computing nodes.
* **Advanced Assembly:** Pushing beyond basic arithmetic in assembly to write algorithms that handle larger datasets would provide a stronger test of memory management skills.
* **Hardware Simulation:** Utilizing more advanced architecture simulation tools (like Logisim or SPIM) to visually trace data paths and pipeline hazards would further solidify the theoretical concepts learned in class.
