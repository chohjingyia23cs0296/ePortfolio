# Human-Computer Interaction (Year 1, Semester 2)

## Overview
Welcome to the `Human Computer Interaction` directory of my ePortfolio. This folder documents the coursework, assignments, and comprehensive projects completed during my second semester. This module explores the intersection of human psychology and software design, focusing on creating intuitive, accessible, and user-centric digital interfaces.

## Objectives
* To understand core UI/UX design principles and user-centered design (UCD) methodologies.
* To analyze user requirements and translate them into functional design solutions.
* To develop proficiency in wireframing and prototyping using industry-standard tools.
* To apply usability testing techniques to evaluate and iterate on digital interfaces.

## What I Did
Throughout this module, I transitioned from theoretical design principles to practical interface creation. The coursework was structured to guide us through the entire design lifecycle, from initial research in the assignments to comprehensive prototyping across the four project phases. I actively utilized prototyping tools like Figma to design a Healthcare App, focusing on clean layouts, illustration integration, and intuitive mobile navigation. Additionally, I contributed to the deployment of a project portfolio site using Wix to document our team's "Future Bright" initiative, ensuring the web presence was as user-friendly as the applications we were designing.

## Key Learning Outcomes
1. **User-Centered Mindset:** Learned to approach software development from the end-user's perspective, prioritizing accessibility and minimal cognitive load.
2. **Prototyping Proficiency:** Mastered the workflow of turning abstract requirements into high-fidelity, interactive digital prototypes using tools like Figma.
3. **Web Deployment:** Gained practical experience in setting up and structuring user-friendly informational websites via platforms like Wix.
4. **Empirical Iteration:** Acquired the skills to evaluate interfaces critically and iterate on designs based on structural heuristics and visual hierarchy.

## Reflection
Studying Human-Computer Interaction provided an incredibly valuable shift in perspective for my path in Data Engineering. While my primary focus involves building robust data pipelines, cloud infrastructure, and structuring SQL/NoSQL databases, this course highlighted that even the most powerful backend system loses value if the end-user cannot easily interpret the final output. 

This UI/UX discipline directly influences how I approach data visualization today. When building complex Power BI or analytics dashboards, I now actively apply HCI principles—such as proper visual hierarchy, minimizing clutter, and providing consistent navigation cues—to ensure that stakeholders can extract meaningful insights instantly and effortlessly.

## Possible Improvements
* **Advanced Interactive States:** If I were to iterate on the Figma Healthcare App, I would focus on creating more complex, data-driven micro-interactions rather than simple page transitions to simulate a more authentic software environment.
* **Broader Accessibility Testing:** Future projects should place a higher priority on accessibility standards (such as WCAG), actively testing color contrast limits and screen-reader compatibility during the initial wireframing phases.
* **A/B Testing Methodologies:** Incorporating quantitative A/B testing on alternative layout concepts would provide stronger statistical evidence to back up final UI layout selections.
