# System Analysis and Design (Year 1, Semester 2)

## Overview
Welcome to the `System Analysis and Design` directory of my ePortfolio. This folder documents my coursework for the SECP2613 module completed during Semester II. The central focus of this directory is the comprehensive analysis and design documentation for the KADA Management Information System. This module bridges the gap between identifying business requirements and mapping out the technical blueprints required to build functional software.

## Objectives
* To model system processes using Logical and Physical Data Flow Diagrams (DFDs).
* To map data interactions utilizing CRUD Matrices and Event Response Tables.
* To design comprehensive system architectures and structural hierarchy charts.
* To develop user interface wireframes for both input and output system interactions.

## What I Did
For this module, I collaborated with my team to completely map out the architecture for the KADA Management Information System. We analyzed the operational requirements for managing memberships, processing loan applications, and generating financial reports. We translated these requirements into structured Logical DFDs to outline the theoretical data flow between system entities. From there, we advanced to Physical DFDs, defining exact system partitioning to separate automated and manual processes. We also created a detailed CRUD matrix to track data creation, reading, updating, and deletion across the system's databases. Finally, we designed the overarching system architecture and drafted visual wireframes to simulate the user interface experience for both members and administrators.

## Key Learning Outcomes
1. **Process Modeling:** Developed strong proficiency in visualizing complex data routing using leveled Data Flow Diagrams.
2. **Data Lifecycle Management:** Learned to meticulously track how specific data entities (like membership or loan details) are handled throughout the system using a CRUD matrix.
3. **System Partitioning:** Gained practical experience in dividing a large, monolithic system into manageable, distinct sub-processes and control engines.
4. **Interface Conceptualization:** Bridged the gap between backend data processing logic and frontend user interaction through detailed wireframing of forms and dashboards.

## Reflection
This project was highly instrumental in shaping my approach to building robust data pipelines. Mapping out the exact flow of the KADA system—from the moment a user submits a registration form to the generation of a final financial report—instilled a strong discipline for architectural planning before writing any code. When structuring relational databases or setting up cloud infrastructure today, the systemic logic cultivated here is heavily utilized. Understanding the CRUD lifecycle ensures that the SQL and NoSQL data storage solutions engineered are fully optimized for how the information will actually be accessed and modified by end-users.

## Possible Improvements
* **High-Fidelity Prototyping:** While the current wireframes effectively outline the structural layout, upgrading them into interactive, high-fidelity prototypes would provide a more accurate feel for the user journey and navigation flow.
* **Cloud Architecture Integration:** The system architecture currently outlines generic application and database layers. Specifying cloud-native components (such as specific container services or managed databases in AWS or Google Cloud) would modernize the architectural blueprint.
* **Security Protocol Documentation:** Future iterations of this design document could explicitly detail the encryption methods or authentication protocols utilized during the login and payment gateway processes to ensure data security.
