# ⚙️ Year 1 — Semester 2

## 📚 Courses
- [Computer Organization and Architecture](./Computer%20Organization%20and%20Architecture)  
  Study of processor design, instruction sets, memory hierarchy and performance considerations with practical exercises.

- [Programming Techniques II](./programming_techniques_II)  
  Continued programming practice building on first-semester foundations; emphasis on more complex data structures and program organization.

- [Human Computer Interaction](./Human%20Computer%20Interaction)  
  Introduction to usability, user-centered design, prototyping and evaluation of interfaces.

- [System Analysis and Design](./System%20Analysis%20and%20Design)  
  Requirements gathering, UML modelling, and designing system specifications and documentation.

## Semester Learning
In Semester 2 I advanced from foundational programming to systems and user-focused design:
- Deeper understanding of computer architecture and how hardware decisions influence software performance.
- Improved programming skills with larger, modular programs and stronger data structure usage.
- Learned user-centred design principles and prototyping techniques for usable interfaces.
- Practiced requirements elicitation, modelling and documentation for software projects.

## Reflection
Semester 2 connected low-level system understanding to higher-level design and usability concerns. Key reflections:
- Knowing architecture helps write more efficient code and reason about performance trade-offs.
- Usability matters: prototypes and user testing reveal assumptions early and prevent rework.
- Systematic requirements and clear models (UML) make complex projects more manageable.
- Continue to document work and create reproducible exercises to support learning and collaboration.
