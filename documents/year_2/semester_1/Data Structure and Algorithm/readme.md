# Data Structure and Algorithm (Year 2, Semester 1)

## Overview
Welcome to the `Data Structure and Algorithm` directory of my ePortfolio. This folder documents my coursework, programming assignments, and projects completed during my second year. This module is a deep dive into how data is organized, stored, and manipulated efficiently within computer memory, forming the backbone of optimized software and data systems.

## Objectives
* To understand and implement fundamental data structures such as linked lists, stacks, queues, trees, and graphs.
* To analyze the time and space complexity of algorithms using Big-O notation.
* To master essential algorithms for sorting, searching, and traversing data.
* To apply appropriate data structures and algorithmic strategies to solve complex computational problems.

## What I Did
Throughout this module, I moved beyond writing functional code to writing *efficient* code. I implemented complex data structures from scratch to understand their internal mechanics and memory management. I spent significant time analyzing the execution speed of different algorithms and determining when to use a hash table versus a binary search tree. Through hands-on programming assignments, I debugged memory allocation issues, optimized loops, and structured data to minimize retrieval times, ultimately compiling these elements into a comprehensive final project.

## Key Learning Outcomes
1. **Algorithmic Efficiency:** Mastered Big-O notation to mathematically evaluate and compare the time and space complexity of different approaches.
2. **Memory Management:** Gained a deep, practical understanding of dynamic memory allocation and pointer manipulation when building linked structures.
3. **Optimal Selection:** Developed the critical thinking skills required to choose the exact right data structure for a specific computational scenario to maximize performance.
4. **Advanced Problem Solving:** Improved my ability to break down complex problems into manageable algorithmic steps, such as using recursion for tree traversals.

## Reflection
This module fundamentally changed how I approach writing code and managing information. While early programming courses focused on making a program work, this class emphasized making it perform efficiently at scale. Understanding the underlying mechanics of structures like B-trees and hash maps provides a massive advantage in the field of Data Engineering. It makes the internal workings of relational SQL databases and NoSQL document stores much less abstract. Knowing exactly how data is indexed and traversed under the hood directly informs how I design, query, and optimize large-scale data pipelines and cloud architectures today.
