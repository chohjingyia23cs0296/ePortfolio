## Overview
This folder documents my learning journey and practical work for the **Database (WBL)** module in Year 2, Semester 1.  
It contains evidence of how I applied database concepts in a work-based learning context, including planning, design, implementation, and testing activities.

## Objectives
- Understand core database concepts such as entities, relationships, keys, and normalization.
- Design structured and efficient relational databases for real use cases.
- Write SQL queries for data definition, manipulation, and retrieval.
- Apply database knowledge in practical/work-based tasks.
- Improve problem-solving and documentation skills through project evidence.

## Key Learning Outcomes
- I can design a relational database structure that matches business requirements.
- I can apply normalization principles to reduce data redundancy.
- I can confidently write SQL for both simple and moderately complex queries.
- I developed better debugging skills when queries returned errors or unexpected results.

## Reflection
This module strengthened my understanding of how databases support real systems and decision-making.  
At first, I found relationship mapping and complex joins challenging, but repeated practice helped me understand the logic more clearly.  
Through work-based tasks, I learned that database accuracy and structure are critical because poor design affects the entire application.  
Overall, this experience increased my confidence in database development and reinforced the importance of careful planning and testing.
