# Object Oriented Programming

## Overview
This folder documents my learning journey in **Object Oriented Programming (OOP)** during Year 2, Semester 2.  
It includes exercises, lab work, and project evidence that show how I applied OOP concepts such as classes, objects, encapsulation, inheritance, polymorphism, and abstraction in practical programming tasks.

## Objectives
- Understand and apply the core principles of OOP.
- Design programs using classes and objects in a structured way.
- Improve problem-solving skills by modeling real-world scenarios in code.
- Practice writing reusable, maintainable, and scalable code.
- Strengthen debugging and testing techniques for OOP-based programs.

## Folder Contents / Evidence
This folder may include:
- **Lecture/Lab Exercises** – practice tasks on classes, constructors, and methods.
- **Assignments/Projects** – larger implementations that combine multiple OOP concepts.
- **Source Code Files** – program files showing class design and object interactions.
- **Screenshots / Output Evidence** – proof of successful compilation and execution.
- **Reports / Notes** – explanations of design decisions and reflections.

## What I Did
- Created classes and objects to represent entities and behaviors.
- Implemented **encapsulation** using access modifiers, getters, and setters.
- Applied **inheritance** to reduce code duplication and create class hierarchies.
- Used **polymorphism** (method overriding/overloading where relevant) to improve flexibility.
- Practiced **abstraction** by focusing on essential features and hiding implementation details.
- Tested and debugged OOP programs to ensure correct functionality.
- Organized code into logical modules/files for clarity and maintainability.

## Key Learning Outcomes
- I can design and implement OOP programs using appropriate class structures.
- I understand how OOP principles improve code organization and reusability.
- I gained confidence in planning program architecture before coding.
- I improved my ability to trace object interactions and debug complex logic.
- I developed better coding discipline through cleaner and more maintainable code practices.

## Reflection
Learning OOP changed how I approach programming problems.  
Instead of writing everything in a single procedural flow, I now think in terms of objects, responsibilities, and relationships.  
This helped me produce code that is easier to read, test, and extend.  
I also learned that good class design is important—poor design can make even small projects difficult to maintain.  
Overall, this topic strengthened both my technical skills and my software design mindset.
