## Semester Learning
Over Semester 2 I advanced core skills in software development, data analysis and systems programming:
- Data programming: developed reproducible data pipelines, practiced data cleaning, exploratory analysis, and visualisation using notebooks and scripted tools.
- Data mining: applied supervised and unsupervised learning workflows, translated domain questions into model experiments, and documented a project from dataset selection to results.
- Object-oriented design: implemented class-based designs, practiced decomposition, encapsulation and reusable component development across multiple lab assignments and a semester project.
- Systems fundamentals: gained hands-on experience with OS concepts—memory allocation strategies, process scheduling and low-level debugging—through lab assessments and a collaborative project.
- Tooling & reproducibility: used Jupyter notebooks, organized assignments and lecture material, and documented steps to reproduce experiments and lab results.

## Reflection
Semester 2 solidified practical foundations that connect data work with reliable software engineering and system-level thinking. Working through data analytic assignments and a data-mining project improved my ability to translate analytical questions into reproducible code and experiments. OOP labs reinforced the importance of clear abstractions and modular design, which made later projects easier to test and extend. Operating System coursework highlighted how lower-level resource management (memory, processes) affects application behaviour and performance—an insight that informed more careful design when developing data-processing and simulation code.

Key takeaways:
- Structure and reproducibility matter: notebooks are great for exploration, but scripts and clear documentation are necessary for reproducible results.
- Start small and iterate: build minimal, testable components (OOP + unit tests) before integrating into larger systems.
- Measure and validate: in data projects, always validate preprocessing steps and measure model performance against clear baselines.
- Systems awareness improves application design: understanding OS-level behaviour helps write more robust, resource-aware programs.
