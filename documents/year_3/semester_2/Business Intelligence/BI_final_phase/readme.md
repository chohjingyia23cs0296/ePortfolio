# Business Intelligence — Final Phase

## Overview
This folder contains the final-phase deliverables for the Business Intelligence project completed in Semester 2, Year 3. The work covers the ETL/workflow design, data analysis, and interactive dashboard deliverables together with the final report.

## Objectives
- Build a repeatable ETL workflow to prepare and transform project data.
- Analyze key business metrics and produce actionable insights.

- Document findings and recommendations in a final project report.

## What I Did
- Assessed the raw data sources and documented data quality issues.
- Designed and implemented an Alteryx workflow (Phase 2) to perform extraction, cleaning, joins and feature preparation for analysis.
- Performed exploratory data analysis to identify trends, outliers and the most relevant business metrics.
- Built visualisations and a consolidated dashboard to present KPIs and support decision-making; exported screenshots are in BI dashboard.pdf.
- Composed the final project report summarising methodology, findings, conclusions and recommendations (see BI Project final phase.pdf).

## Key Learning Outcomes
- Hands-on experience designing an ETL pipeline and automating data preparation using Alteryx.
- Improved ability to translate business questions to measurable metrics and visualisations.
- Practical skills in exploratory data analysis and identifying data quality problems.
- Experience in communicating analytical findings clearly through a written report and dashboard visuals.

## Reflection
This final phase brought together the technical pipeline work and the business-facing deliverables. The Alteryx workflow made the data preparation reproducible and substantially reduced manual processing time. The dashboard helped synthesise insights for stakeholders and highlighted both strengths and areas for improvement in the underlying data.

Challenges included handling inconsistent source data and deciding which derived metrics best reflected business outcomes. Working through these constraints improved my data-cleaning judgement and reinforced the importance of documenting assumptions.

## Possible Improvements
- Add automated testing/validation steps to the ETL workflow to catch schema changes or unexpected nulls earlier.
- Enhance the dashboard with interactive filters and drill-through capability for deeper stakeholder exploration.
- Consolidate code and workflow versioning so that each dashboard build is linked to a specific workflow commit or snapshot.
- Expand the report with more scenario analysis and sensitivity checks to stress-test recommendations.
