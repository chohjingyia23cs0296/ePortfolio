## Semester Learning
Over the semester I consolidated and expanded practical skills across software development, data engineering and analytics:
- Mobile & frontend engineering: cross-platform app design and state management with Flutter; platform integration (Firebase, push notifications).
- Data engineering pipelines: ETL/ELT practices, dataset collection, cleaning, transformation and storage for downstream analysis.
- High-performance data processing: asynchronous I/O, concurrency control, batching, resource profiling (time/memory) and optimization strategies for large-scale scraping and processing.
- Business intelligence & analytics: preparing datasets for analysis, performing EDA, visualising results and assembling production-ready reports/dashboards.
- Reproducibility & tooling: using notebooks, dependency manifests, scripted workflows and documentation to make results reproducible and shareable.
- Software engineering practices: testing, logging, documentation, and considerations for deployable artefacts (containerization, CI/automation).

## Reflection
This semester balanced applied software development with data-focused engineering. Application Development strengthened my ability to design and ship interactive mobile experiences and manage platform-specific constraints. Business Intelligence taught disciplined data preparation and how to translate cleaned data into actionable insights and reports. High Performance Data Processing deepened my appreciation for engineering trade-offs when collecting and processing large datasets—optimizations such as async IO and batching yield big gains but require robust error handling and careful politeness (rate-limiting) when scraping.

Across courses, the most valuable lessons were:
- Design for reproducibility: document steps, pin dependencies and include scripts/notebooks so work can be re-run and validated.
- Measure before/after: profiling (time, CPU, memory) is essential to make optimization decisions that matter.
- Balance performance with responsibility: high throughput is useful, but respecting target systems (rate limits, robots.txt) and legal/ethical constraints is mandatory.
